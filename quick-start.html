Quick Start

Get Morse running on testnet in under 10 minutes. You will deploy the contracts, create your first publication, add a collection, and publish your first content entry.

Prerequisites

Make sure you have the following installed before proceeding:

ToolVersionPurpose
Sui CLILatestBuild and deploy Move contracts
Bunv1.2+Runtime for CLI and SDK
Taskv3+Task runner for contract workflows

You will also need a funded Sui testnet wallet. Use sui client faucet to get test SUI tokens.

1. Clone the Repository

bash 
git clone https://github.com/TheDivic/morse-dcms.git
cd morse-dcms/morse-contracts

2. Build & Test Contracts

Ensure you are on the Sui testnet environment before building. The Move.toml is configured to target testnet by default.

bash 
# Build the Move package
task build

# Run the full test suite
task test

# Run a single test by name
sui move test test_new_publication

3. Deploy to Testnet

⚠️
Funded wallet required
Your active Sui wallet must have testnet SUI tokens. Run sui client faucet to request tokens from the faucet.
bash 
task publish

After publishing, the package address and shared object IDs are saved to Published.toml. You will need the PublicationRegistry object ID for subsequent transactions.

Local Development (Optional)

If you want to iterate without touching testnet, use the localnet workflow:

bash 
# Terminal 1  keep running
task localnet

# Terminal 2  wait ~60s for coins to arrive
task faucet

# Publish ephemerally (does not update Published.toml)
task publish:local

# Clean up and switch back to testnet
task clean
task switch:testnet
ℹ️
Chain ID mismatch?
Each fresh localnet gets a new chain ID. If publish:local fails with a chain-id mismatch, run task clean first to remove the stale Pub.local.toml.

4. Create Your First Publication

Once deployed, interact with the contracts using Programmable Transaction Blocks (PTBs) via the Sui CLI. The new_publication function returns three objects share the publication and transfer the capabilities in the same PTB.

move 
// In your PTB (Programmable Transaction Block):

// Step 1  create publication, returns (Publication, OwnerCap, PublisherCap)
let (publication, owner_cap, publisher_cap) = publication::new_publication(
  &mut registry,
  ctx,
  b"My Blog".to_string(),
  b"my-blog".to_string(),  // unique slug: lowercase, alphanumeric, hyphens
);

// Step 2  share the publication so publishers can write to it
publication::share_publication(publication);

// Step 3  transfer caps to your wallet
transfer::transfer(owner_cap, sender);
transfer::transfer(publisher_cap, sender);

5. Add a Collection

Choose a storage mode based on your content type. Use Blob mode for articles and posts; Quilt mode for site pages and structured configs.

move 
// storage_mode: 0 = blob, 1 = quilt
publication::create_collection(
  &mut publication,
  &publisher_cap,
  b"articles".to_string(),
  0,   // STORAGE_MODE_BLOB
  ctx,
);

6. Publish Your First Entry

Upload your content to Walrus first to get a Blob object, then register it on-chain:

move 
// blob is a Walrus Blob object (must be deletable)
let entry_id = publication::add_entry_to_collection(
  &mut publication,
  &publisher_cap,
  b"articles".to_string(),
  b"Welcome Post".to_string(),   // entry name
  &blob,                              // Walrus Blob object
  option::none(),                   // no QuiltPatchId for blob mode
  b"application/json".to_string(), // MIME content type
  false,                             // not encrypted
  entry::access_policy_public(),   // ACCESS_PUBLIC = 0
  option::none(),                   // no Seal identity
  ctx,
);
// entry_id is the stable monotonic ID assigned (0, 1, 2, ...)
✅
You're live on-chain
The entry is now registered. Its public_head points to revision 0, making it immediately readable. Any indexer watching the chain will pick it up.
