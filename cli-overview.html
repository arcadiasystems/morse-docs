CLI Overview

morse-cli is the command-line client for the Morse decentralized CMS. It signs with a locally encrypted key, stores content on Walrus, and encrypts private entries with Seal: the same protocol surface as the SDK, driven from your terminal. Runs on Node or Bun, pre-release on testnet.

💻
Published on npm · testnet only
Install @arcadiasystems/morse-cli from npm. Addresses are baked in for testnet; mainnet support arrives once the contracts are frozen at v1.0.0.

Install

Runs under Node (>= 18) or Bun (>= 1.2). The published bin is a Node-targeted bundle, so neither runtime is required to install, only to run.

bash 
npm i -g @arcadiasystems/morse-cli     # or: bun add -g, pnpm add -g
morse --help

# run without installing:
npx @arcadiasystems/morse-cli --help   # or: bunx @arcadiasystems/morse-cli

You also need a funded Sui testnet address for gas, and testnet WAL for Walrus storage when adding content (faucets: SUI, WAL).

The friendly flow

Select an active publication and collection once; commands then default to them, so you never paste a 64-character object id. publication create and collection create auto-select the new object.

bash 
morse config add testnet --network testnet   # create a profile (becomes default)
morse account import                          # import a key (hidden prompts)
morse publication create --name "My Blog" --slug my-blog   # becomes active
morse collection create posts                 # becomes active
morse entry add hello --file post.md          # no ids needed; prints a view link
morse entry read 0                            # read the content back

Active context

The active publication and collection live in the current profile. Override per command with -P, --publication <slug|id> and -C, --collection <name>; flags always win over the context. Slugs resolve against publications owned by the active account.

  • morse use <slug|id> [collection]: set the active publication and optional collection.
  • morse use --clear: clear them.
  • morse status: show the active profile, account, publication, and collection.

Security model

Private keys are stored in an encrypted keystore (scrypt + AES-256-GCM) and unlocked by a password.

  • Keys are never accepted as command-line flags (they would leak via ps, /proc, and shell history). They come only from an interactive hidden prompt, the MORSE_PRIVATE_KEY env var, or the encrypted keystore.
  • Keystore files are written chmod 600; group- or world-readable key files are refused on read.
  • Keys are never printed or logged, including under --debug. The sole exception is account export, which deliberately reveals a key: interactive-only, requires confirmation, unavailable in --json.
  • For CI, set MORSE_PRIVATE_KEY (raw key, never persisted) and MORSE_KEYSTORE_PASSWORD to skip prompts.

Configuration

Settings resolve with the precedence flags > MORSE_* env > config file > defaults. The config file lives under $XDG_CONFIG_HOME/morse (default ~/.config/morse); encrypted keystores live in keystores/ beside it.

VariableOverridesNotes
MORSE_PROFILE--profileProfile to use.
MORSE_NETWORK--networktestnet or localnet.
MORSE_RPC_URL--rpcSui RPC URL override.
MORSE_ADDRESS(no flag)Active account address.
MORSE_PUBLICATION-PActive publication id.
MORSE_COLLECTION-CActive collection name.
MORSE_PRIVATE_KEY(no flag)Raw Bech32 secret key. Highest-priority key source; never persisted.
MORSE_KEYSTORE_PASSWORD(no flag)Keystore password for non-interactive unlock.

Output & scripting

  • Human output goes to stdout; progress, warnings, and prompts go to stderr. Redirecting stdout yields only the result.
  • --json emits a single JSON document on stdout. bigint values (e.g. gasUsedMist) are encoded as decimal strings and byte arrays (sealId, quilt patch ids) as 0x-hex.
  • Color is emitted only to a TTY and honors NO_COLOR / FORCE_COLOR.

Exit codes

CodeMeaning
0Success
1Generic error (contract abort, uncategorized)
2Usage error (bad flags/args, invalid id/slug/address, declined confirmation)
3Not found
4Auth or permission failure (keystore unlock, unauthorized, no-access)
5Network or transport failure
